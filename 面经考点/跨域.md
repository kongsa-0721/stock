###    跨域是什么意思？ 

浏览器在解析ajax请求的时候  必须要满足同源策略 浏览器的请求路径 和 Ajax的网址之间 满足同源策略 。跨域只存在于浏览器 不存在于服务器
同源策略/SOP（Same origin policy）是一种约定，由Netscape公司1995年引入浏览器，它是    浏览器最核心也最基本的安全功能，如果缺少了同源策略，浏览器很容易受到XSS、CSFR等攻击。所谓同源是指”协议+域名+端口”三者相同，即便两个不同的域名指向同一个ip地址，也非同源。

- CORS 支持所有类型的 HTTP 请求，是跨域 HTTP 请求的根本解决方案
- JSONP 只支持 GET 请求，JSONP 的优势在于支持老式浏览器，以及可以向不支持 CORS 的网站请求数据。
- 不管是 Node 中间件代理还是 nginx 反向代理，主要是通过同源策略对服务器不加限制。
- 日常工作中，用得比较多的跨域方案是 cors 和 nginx 反向代理
-  无法读取非同源网页的cookie、localstorage等 
-  无法接触非同源网页的DOM和js对象 
- 无法向非同源地址发送Ajax请求

### 怎么解决跨域？ 

1.jsonp 一些标签是不存在跨域问题的   原理是src不受跨域影响，所以让后台返回一个js文件，js里的代码包含数据。一般就是前端写个函数，把函数名告诉后端，后端返回一段js文件，里边内容为执行这个函数并传参，参数就是要返回的数据 。 用jsonp抓取到的资料并不是json 而是任意的js代码 用js 编译器执行。动态生成script标签 通过src加载

```js
jsonp先注册一个函数 
<script>
	function f(data){
		alert(data)
	}
</script>
//设置第二个标签 请求数据 
<script src="http://localhost:8080?callback=f"></script>

在后台
把callback取出来 接收一下
var func = req.query.callback;
res.send(func+"('你好')")

返回给前端就拼合成
f('你好') 就执行这个代码了
```
